Quando gostaria de usar um repositório remoto em vez de manter o trabalho local?

Quando estou trabalhando em um projeto que contém muitas pessoas e preciso que essa informação possa ser compartilhada, ou quando preciso usar mais de um computador 

Por que você prefere efetuar pull de alterações manualmente do que deixar o Git atualizar-se automaticamente com o seu repositório remoto?

Ao efetuar o pull eu garanto que tanto o repositorio remoto, quanto o repositorio local estão iguais, isso facilita meu trabalho pois além de eu ter duas estações de trabalho prontas uma para quando eu ter internet e outra para quando eu nao estiver não à risco de conflitos entre repositorios.


Descreva as diferenças entre bifurcações, clones e branches. Quando você usaria um e não os outros?

bifurcação é algo especifico do github, muito parecido com um clone do git entretanto a diferença é que ele faz um fork diretamente para sua pagina no github, clone é o ato de clonar genuinamente um repositorio e caso voce use de um git remote ele estará apontando diretamente de onde vc realizou a clonagem, o branche nada mais é que uma nova saida para realização de atividades extras.

Qual é o benefício de ter uma cópia do último estado conhecido do remoto armazenado localmente?

O beneficio é que os dois repositorios estarão up-to-date, ou seja, se os dois estão sincronizados então a probabilidade de ocorrer alguma falha frente a commits de varias pessoas diferentes é menor

Como você colaboraria sem usar Git ou GitHub? O que seria mais fácil e mais difícil?

eu colaboraria usando Github, no começo é complicado visto a dificuldade de alguns conceitos e entender toda a base, entretanto quando você o entende a ideia fica simples e o ganho de tempo é bem maior. colaborar sem usar o git é uma opção, mas quando vc trabalha com codigo essa opção não é adequada.

Quando você gostaria de fazer alterações em um branch separado, e não diretamente no master? Quais são os benefícios de cada abordagem?

Quando eu preciso do feedback de outras pessoas para poder confirmar minhas alterações, as vezes erros passam despercebidos ou algo que eu fiz não necessariamente esta correto, antes de confirmar uma alteração no master que pode gerar um gargalo, eu peço opiniões dos outros colaboradores e ai sim faço um merge.
